## 클라우드 컴퓨팅이란?

종량제 요금으로 인터넷을 통해 IT 리소스를 온디맨드로 제공하는 것

온디맨드 : 필요한 만큼 리소스를 사용하는 것

종량제 요금 : 고객이 실제로 사용한 클라우드 리소스의 양에 따라서만 비용을 지불하는 과금 방식

## 클라우드 컴퓨팅의 이점

1. 종량제 기능
    
    클라우드 컴퓨팅을 사용하면 고정 비용을 가변 비용으로 바꿀 수 있다. 온프레미스 방법으로 구축하게 되면 여러 부분에서 비용이 많이 나가게 된다. 따라서 종량제가 클라우드 컴퓨팅의 이점이 될 수 있다.
    
2. 거대한 규모로 경제적 이익을 얻을 수 있다.
    
    클라우드 컴퓨팅을 제공하기 위한 하드웨어를 거대한 규모로 가지기 때문에 저렴하게 얻을 수 있다. 때문에 절감된 비용은 고객이 얻을 수 있다.
    
3. 사용할 용량을 정하지 않아도 된다.
    
    필요한 리소스를 프로비저닝 하면되며, 스케일링 메커니즘을 사용하여 수요에 맞추어 스케일 업, 다운을 할 수 있다.
    
    프로비저닝 : 사용자 요구에 맞게 서버, 스토리지 등을 자동으로 제공하는 모든 과정
    
4. 속도, 민첩성 향상
    
    프로비저닝 및 해제에 드는 시간을 줄이고 최적화에 주력할 수 있다.
    
5. 데이터 센터 운영, 유지 관리에 비용 투자가 필요하지 않다.
    
    서버 랙 설치, 전원 공급 등 리소스를 허비하는 대신 다른 부분에 더 신경쓸 수 있다.
    
6. 단 시간 내, 전세계에 배포할 수 있다.

### 이점 정리

1. 고정비용을 가변 비용으로 대체할 수 있다.
2. 다양한 규모로 비용 절감을 야기할 수 있다.
3. 사용할 용량을 추정하지 않아도 된다.
4. 신속하게 많은 영역으로 서비스를 배포할 수 있다.
5. 데이터 센터 유지 관리를 하지 않아도 된다.

## AWS 글로벌 인프라

하나의 데이터 센터에 모든 리소스를 보관하는 것은 위험하다. 따라서 고가용성과 내결함성이 필요하다.

- 고가용성 : 가동 중지 시간을 최소화하면서 애플리케이션에 계속 액세스할 수 있도록 보장하는 것으로, 한 구성 요소가 실패해도 다른 구성 요소가 빈 자리를 채우기 때문에 서비스가 계속 실행될 수 있다.
- 내결함성 : 여러 구성요소가 실패했을 때 계속 동작할 수 있도록 설계하는 것이다.

AWS가 전 세계에 리전을 운영하는 이유는 고가용성과 내결함성을 고려하여 설계를 했기 때문이다. 이렇게 시스템이 중단되지 않도록 한다.

### 글로벌 인프라

**AWS 리전** : 데이터 센터 그룹이 포함된 전 세계의 실제 위치이다. 이러한 데이터 센터 그룹을 가용 영역이라고 한다.

**가용 영역** : 중복 전력, 네트워킹, 연결 기능을 가진 1개 이상의 데이터 센터로 구성된다.

### 글로벌 인프라 고가용성

AWS 인프라는 고가용성, 내결함성을 생각하여 설계되었다. 격리된 리소스로 구성되어 있고, 독립적인 전력, 네트워킹, 연결 기능을 가지고 있다.

### AWS 공동 책임 모델

AWS의 보안 책임자는 AWS와 사용자 둘 모두이다.

사용자는 클라우드 내부의 보안을 책임진다. AWS는 물리적인 네트워크, 하이퍼바이저 계층을 책임진다.
