## AWS User Data를 무조건 써야 하는 경우
1. AMI를 구성하지 말아야 할 때.
2. EC2 부팅 시 자동으로 ~가 실행되어야 할 때
3. 오토 스케일링을 위한 설정이 필요할 때

## User Data를 구성하기 위한 `3단계 분석법`
1. `어떤 소프트웨어`를 사용하는가?
   - Linux 2, apache 2 패키지 등등
2. 파일 `내용이나 경로`는 무엇인가?
   - index.html, "Hello Cloud Computing" 등등
3. `외부`에서 가져와야 하는 `리소스`가 있는가?
   - /dist.png 입력 시 이미지 나타내기 등

## 정리
기능경기대회에서 userdata 입력 시간을 줄이려면, 일단 `어떤 소프트웨어를 사용하는지` 명확히 알아야 한다.
-y 옵션은 무조건 있어야 한다. 스크립트 중간에 직접 "yes"를 누를 수 없기 때문이다. 예를 들어 yum install -y" 처럼 강제 설치 옵션을 반드시 넣어야 한다.

무언가를 print 해야 할 때는 대소문자, 띄어쓰기를 주의 해야 한다.

과제에 따라 권한이 다르기 때문에 스크립트 안의 "aws s3 cp"와 같은 명령어가 들어간다면, 해당 인스턴스에 반드시 s3 읽기 권한이 있는 IAM 역할이 부여되어야 작동된다.

스크립트 root 권한으로 실행되기 때문에 파일 경로는 가급적 `절대 경로`로 작성하는 것이 좋다.
